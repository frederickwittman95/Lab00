By the end of this lab, you will have a code repository on github that contains the following documents:

  • README.md
  • HelloGithub.cpp
  • HelloGithub.py
  • <yourname>_HelpSheet.txt

During the course of this lab you will:

  • Clone a repository
  • Create a branch
  • Create a file within a branch
  • Commit a branch
  • Create and accept a pull request

Getting Started with Github

We will use the website github.com to complete this lab, but you could use the command line interface (CLI) or desktop program (desktop.github.com), if you prefer. Although the GIT software system is primarily for source code version control, we are using GitHub for distributing program files through repositories.

Preparation

Go to http://guides.github.com/activities/hello-world/  and read through the entire page. This is quite similar to what you must do for this lab so follow along closely, although you will not be creating a new repository but rather forking and cloning a class reporitory.

If you do not yet have a Github account then this is the time to create one.

  1. Log in to http://classroom.github.com using your github user account
  2. Go to the following page to link to our classroom account: https://classroom.github.com/a/awdbIYCE
  3. Find your email address within the list of available accounts. If yours does not show up, let your instructor and lab assistant know right away.
  4. On click of "Accept this assignment," the lab00-assignment-<your-name&rt; repository will be available in your GitHub account.

Tasks

  1. Clone the Lab00 to a local system, wherever you want to work, such as a student lab computer or your own home computer.
  2. Create a new branch in your Lab00 repository. You could call it "branch-00" or something like that. (Remember, if you make a mistake, you can branch again from "master.")
  3. Click on the edit (pencil) icon for the README.md file and add a sentence describing your previous experience with C++ and Python, signing with your name. (Scroll down to commit changes.)
  4. Run both "Hello Github" programs, one in C++ and one in Python. You will probably need to consult other sources to find out how to do that. Make sure that the input file is in the correct folder. Check that the output file is produced correctly.
  5. Add comments to the source code for each program, one comment or more per line of code, explaining what the code does. Run the programs again to make sure they still work. Add your modified source code files to your repository, using the Upload Files button, and don't forget to commit changes.
  6. Back in your new branch on GitHub.com, create a new text file called <yourname>_HelpSheet.txt that notes these helpful pieces of information:
    • How to comment C++ code
    • How to comment Python code
    • What kind of file has extension ".md"
  7. You will add to this file throughout the semester. Use the Upload Files button to add to your new branch.
  8. Commit and push this branch to your repository. Note the "diffs" (difference in original and files) that show in the comparison.
  9. Next merge your new branch and merge it into the master branch by opening a Pull Request. You have to name it and describe it.
  10. Now click Merge Pull Request. This will update your own master version of the Lab00 files.
  11. Use the @mention feature to notify us when you do this. Our Git names are @? and @robinkhill.
  12. Then approve the Pull Request. Your master branch should now be the same as your new branch with the additional text files.
